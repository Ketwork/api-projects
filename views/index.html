Back End Development and APIs Project Portfolio

KET PATEL

★ Timestamp Microservice

User stories :
  1. I can pass a string as a parameter, and it will check to see whether that string contains either a unix timestamp or a natural language date (example: January 1, 2016)
  2. If it does, it returns both the Unix timestamp and the natural language form of that date.
  3. If it does not contain a date or Unix timestamp, it returns null for those properties.

★ Request Header Parser

User stories :
  1. I can get the IP address, language and operating system for my browser.

★ URL Shortener

User stories :
  1. I can pass a URL as a parameter and I will receive a shortened URL in the JSON response.
  2. If I pass an invalid URL that doesn't follow the valid http://www.example.com format, the JSON response will contain an error instead.
  3. When I visit that shortened URL, it will redirect me to my original link.

★ Excercise tracker

User stories :
  1. I can enter a username on the form and a new user will be created in the database. The JSON response will contain username and ID properties.
  2. I can get a list of all the users at /api/users.
  3. Exercises can be added to the user with the user id. If no date is supplied, the current date will be used.
  4. The response returned from POST /api/users/:_id/exercises will be the user object with the exercise fields added.
  5. I can make a GET request to /api/users/:_id/logs to retrieve a full exercise log of any user.
  6. I will also get count property representing the number of exercises that belong to that user.
  7. I can add from, to and limit parameters to a GET /api/users/:_id/logs request to retrieve part of the log of any user.
